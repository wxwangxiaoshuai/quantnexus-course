## 你不需要预测市场，但你需要识别市场

在 m8l1 中你学到：「趋势跟踪在震荡市亏损，均值回归在单边趋势中亏损」。一个自然的问题是：**怎么知道当前市场是趋势还是震荡？**

这就是市场状态识别（Regime Switching）——不预测未来，只判断现在。

> 关联课程：m8l1「四大策略范式」→ m4l3「ADX/ATR 指标」

## 识别市场状态的三大工具

**工具 1：ADX 趋势强度**

ADX 不判断方向，只判断「趋势的可信度」：

| ADX 值 | 市场状态 | 策略建议 |
|--------|----------|----------|
| < 20 | 震荡市 | 暂停趋势策略，或切换到均值回归策略 |
| 20-25 | 趋势正在形成 | 降低仓位，等待确认 |
| 25-40 | 强趋势 | 趋势策略全仓运行 |
| > 40 | 极端趋势 | 警惕趋势末端（可能即将反转）|

**工具 2：波动率分位数**

用 ATR(20) 在过去 1 年中的分位数判断当前波动环境：

| ATR 分位数 | 市场状态 | 策略建议 |
|-----------|----------|----------|
| < 20% | 低波环境 | 可以适当增加仓位（波动率低 = 风险低）|
| 20-80% | 正常波动 | 维持标准仓位 |
| > 80% | 高波环境 | 降低仓位（波动率聚集——大波动后仍是大波动）|

**工具 3：均线排列**

用 MA5 / MA20 / MA60 的排列判断趋势方向：

- MA5 > MA20 > MA60 → 多头排列（上升趋势）
- MA5 < MA20 < MA60 → 空头排列（下降趋势）
- 三者交叉缠绕 → 震荡市

:::highlight orange
**状态切换策略的风险：状态判断本身可能就是错的**

你判断当前是震荡市，于是关闭了趋势策略——但市场突然开始了一波单边上涨。你不仅错过了趋势，还可能在震荡策略中做空亏了钱。

**双重错误的代价 > 不切换的代价**。

应对策略：
1. **不要频繁切换**——状态判断至少以周为频率，不要日频切换
2. **用确认窗口**——ADX 连续 5 天 > 25 才确认进入趋势状态，避免单日假突破
3. **切换时渐进调整仓位**——不要全仓切，先降 50% 旧策略、加 50% 新策略
4. **保留对冲仓位**——不把所有资金都压在一种状态判断上
:::

## 状态切换的实战框架

以下是一个可操作的框架，你可以直接用在策略管理中：

```
每个周末执行以下检查：

1. 计算所有关注品种的 ADX(14)
2. 计算 ATR(20) 在过去 1 年中的分位数
3. 查看 MA5/MA20/MA60 排列

如果 ADX > 25 且均线多头排列 → 趋势状态：
  - 运行趋势跟踪策略（如 Donchian 突破）
  - 暂停均值回归策略

如果 ADX < 20 且均线缠绕 → 震荡状态：
  - 暂停趋势跟踪策略
  - 运行均值回归策略（如布林带反转）

如果 ATR 分位数 > 80% → 高波状态：
  - 所有策略仓位降到正常的 50%

如果以上条件都不满足 → 中性状态：
  - 维持现有策略不变
  - 不做任何调整
```

## 状态切换的数学检验

在你把状态切换加入策略之前，先做一件事：**回测验证状态切换是否真的改善了绩效**。

很多时候你会发现：
- 状态切换带来的额外收益 ≈ 被状态判断错误造成的损失抵消
- 最简单的做法（始终运行同一个策略）反而更好

**只有当你确认状态切换在样本外回测中确实改善了 Sharpe 时，才把它加入实盘策略。**
