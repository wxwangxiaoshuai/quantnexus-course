## 项目一总览：CTA 趋势跟踪策略——从信号到实盘的全流程

> ⏱ **关于 duration 标签**：本模块（M12-M14 实战项目）的「预计时长」仅含正文阅读。含代码实践、系统操作、回测验证的课节，实际耗时可能是标注值的 **1.5-2 倍**——别因耗时超出而自我怀疑。

这是你的第一个生产级项目。你将用 QuantNexus 系统的真实功能，完成一个完整的策略研发流程：

```
数据中心下载数据 → 策略模板改造 → 三维回测验证 → 完整风控配置 → 模拟盘上线
```

**品种**：螺纹钢 rb（期货市场流动性最好的品种之一）
**策略类型**：唐奇安通道突破（Donchian Channel Breakout）——趋势跟踪的经典实现
**时间范围**：2023-01-01 ~ 2025-12-31（3 年日线数据）

**你将学到**：
- 如何在数据中心下载和管理历史数据
- 如何检查数据质量（缺失值、异常点、覆盖完整性）
- 如何将原始数据导出 CSV 做离线分析

> 关联课程：m3l1「Tick 数据」→ m3l2「K 线」→ m3l5「数据管线与质量工程」

## 步骤 1：打开数据中心

在 QuantNexus 侧边栏点击「数据中心」（/data），你会看到：

- **数据下载 Tab**：按品种 + 时间范围下载历史 K 线
- **数据覆盖 Tab**：查看各品种的数据覆盖情况（日历热力图）
- **任务历史 Tab**：查看下载/修复任务的状态

## 步骤 2：下载螺纹钢日线数据

在「数据下载」Tab 中填写以下参数：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 合约代码 | `rb` | 螺纹钢品种代码 |
| 交易所 | `SHFE` | 上海期货交易所 |
| K 线周期 | `1d` | 日线（每个交易日一根 K 线） |
| 起始日期 | `2023-01-01` | 3 年前 |
| 结束日期 | `2025-12-31` | 最近 |
| 数据源 | `shinny` | 或 rqdata（取决于你的配置） |

点击「下载」按钮后，系统会创建一个异步下载任务。你可以在「任务历史」Tab 中观察进度：
- `pending` → `running` → `completed`

> 3 年日线数据大约 720 个交易日，下载通常 1-2 分钟完成。

:::highlight orange
**数据质量是量化策略的第一道防线**。下载完成后，不要急着做策略——先检查数据质量。以下三个指标是职业量化研究员的必修课：

1. **缺失 K 线**（gap_count）：3 年应有 ~720 个交易日，缺失 < 5 天为正常（节假日已排除）
2. **价格异常点**（price_spikes）：单日涨跌幅超过涨跌停板的 1.2 倍 → 可能是数据错误
3. **零成交量日期**（zero_volume_days）：日线成交量不应为零（除非合约已退市）

这些检查对应 m3l5「数据管线与质量工程」——脏数据 = 带 bug 的代码上生产，回测结果再漂亮也是假的。
:::

```python
# 使用 QuantNexus API 导出数据（在 Python 环境中运行）
import requests

# 方式 1：通过 API 同步导出 CSV
url = "http://localhost:8000/api/data/export"
params = {
    "symbol": "rb",
    "interval": "1d",
    "start": "2023-01-01",
    "end": "2025-12-31",
}
# 需要认证 token，在 Settings 页面获取
headers = {"Authorization": "Bearer YOUR_TOKEN"}
response = requests.get(url, params=params, headers=headers, stream=True)
with open("rb_1d.csv", "wb") as f:
    for chunk in response.iter_content(chunk_size=8192):
        f.write(chunk)

# 方式 2：在数据中心页面点击「导出」按钮直接下载 CSV

# 加载后做基本探索
import pandas as pd
df = pd.read_csv("rb_1d.csv", parse_dates=["datetime"], index_col="datetime")
print(f"数据范围: {df.index[0]} ~ {df.index[-1]}")
print(f"交易日数: {len(df)}")
print(f"缺失日期: {pd.bdate_range(df.index[0], df.index[-1]).difference(df.index)}")
print(f"价格范围: {df['close'].min():.0f} ~ {df['close'].max():.0f}")
```

## 步骤 3：数据覆盖可视化

在「数据覆盖」Tab 中，输入 `rb` 和 `1d`，选择 2023-01-01 ~ 2025-12-31，点击查询。

系统会显示一个日历热力图——每个格子代表一个交易日，绿色表示有数据，红色表示缺失。

**预期结果**：3 年数据完整覆盖，偶尔有 1-2 天缺失（可能是交易所临时休市）。

## 步骤 4：数据质量检查

在「数据质量」Tab 中，同样输入参数，系统会返回：
- `gap_count`：缺失 K 线数量
- `price_spikes`：价格异常点列表
- `zero_volume_days`：零成交量日期

> 如果发现异常点，可以用「数据修复」功能重新下载覆盖异常数据。

## 为什么这一步很重要？

你可能会想「直接做策略不行吗？」——不行。真实数据的问题比你想象的多：
- 交易所临时休市（台风、系统故障）
- 数据商传输丢包（某几天 K 线缺失）
- 撮合引擎 bug 产生极端价格（如 2020 年 WTI 负油价）

职业量化研究员在数据清洗上花的时间，通常占策略研发总时间的 30%。这一步不做，后面的回测结果全是垃圾进垃圾出。
