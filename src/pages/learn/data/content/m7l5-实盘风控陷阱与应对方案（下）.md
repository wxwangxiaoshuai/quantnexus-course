## 陷阱 4：流动性枯竭

**场景**：你持仓 50 手某品种，盘口显示卖一挂单只有 5 手。你想止损平仓时，吃掉卖一（5 手）、卖二（3 手）、卖三（2 手）...价格被你的平仓单推着往下走——这就是「自我滑点」。更糟的是，市场恐慌时流动性瞬间消失（闪崩），任何价位都没有买单。

**应对方案**：
- **事前流动性检查**：开仓前检查该品种的日均成交量和盘口深度。经验法则：你的目标持仓量不超过日均成交量的 1%。
- **分批次平仓**：用时间加权平均价格（TWAP）或成交量加权平均价格（VWAP）算法，把大单拆成小单，在不同时间点分批执行。
- **冰山委托**：只显示委托总量的一小部分（如 10%），其余隐藏，避免大单引起市场恐慌。
- **流动性储备**：在策略中预留部分资金应对极端流动性事件，不在恐慌时被迫平仓。

## 陷阱 5：系统/网络故障

**场景**：你的策略正在运行，突然网络断了 3 分钟。这 3 分钟里行情剧烈波动，你的止损单没有发出去。重连后发现仓位已经从盈利 2% 变成亏损 8%。

**应对方案**：
- **服务端止损（硬止损）**：在经纪商/交易所端设置一个「硬止损」条件单（服务器端执行，不依赖本地），即使你的程序完全崩溃，止损仍然生效。
- **心跳检测**：策略每 N 秒发送心跳，如果超过 M 秒未收到心跳 → 触发应急流程（平掉所有持仓、暂停新开仓）。
- **冗余连接**：双网络路径（主 + 备用 4G/5G）、双服务器（主 + 热备）。
- **本地风控进程**：在策略进程之外运行一个独立的风控守护进程，监控持仓和风险敞口——策略挂了它还在。

## 陷阱 6：过度交易（Overtrading）

**场景**：策略一天产生 50 个信号，每个似乎都不错。交易员（或程序）照单全收，一天下来赚了 200 元，手续费花了 300 元。

**核心公式**：
$$
\text{净收益} = \text{毛收益} - N_{\text{trades}} \times \text{单笔成本}
$$

**应对方案**：
- **成本模型前置**：在策略中明确定义每笔交易的预期成本（手续费 + 滑点估算），只有信号强度的预期利润超过 2 倍成本时才执行。
- **信号过滤**：增加信号的置信度阈值——例如只在 MACD 金叉 + 放量确认时才入场，单纯金叉过滤掉。
- **交易频率上限**：设置单日/单小时的最大交易次数硬限制。
- **手续费后回测**：所有回测必须包含手续费，否则回测收益毫无意义。

## 陷阱 7：心理陷阱与程序化风控

量化交易的一大错觉：以为用了程序就不会受情绪影响。事实是：你在看盘时看到策略连续亏损 8 次，还是会想「关掉策略手动干预」。

**常见心理陷阱**：

| 陷阱 | 表现 | 风险 |
|------|------|------|
| 干预冲动 | 策略连亏时手动停止，错过了后续的盈利信号 | 破坏了策略的长期期望值 |
| 择时心理 | 等「更好的时机」才开策略，但永远找不到完美时机 | 空仓等待时市场已经走了 |
| 过度自信 | 回测表现好后加大仓位，远超凯利给出的最优比例 | 一次黑天鹅就爆仓 |
| 近期偏差 | 最近 3 个月表现好就往里加钱，最近 3 个月差就撤资 | 高处买入、低处卖出，反向操作 |
| 结果导向 | 只看盈亏判断策略好坏，不看过程是否按规则执行 | 好的规则可能被错误的信号淘汰 |

**应对方案 — 程序化纪律**：

1. **策略 = 宪法**：回测通过的策略规则就是宪法，除非样本外验证有明确的退化信号，否则不修改、不干预、不择时。
2. **盈亏日志**：记录每笔交易「是否按规则执行」而不仅是盈亏。关注过程质量，而非结果好坏。
3. **资金冻结期**：任何人工干预（加仓/减仓/关策略）后，锁定 30 天不允许再次干预。
4. **纪律红绿灯**：策略回撤 < 10%（绿灯正常）→ 10%-20%（黄灯，检查但不干预）→ > 20%（红灯，暂停并样本外复盘）。

## 陷阱 8：黑天鹅与肥尾风险

**场景**：你基于「日收益率服从正态分布」设计了仓位和止损。但实际上金融市场的收益率分布有明显的「肥尾」——3 个标准差以上的事件发生的概率远超正态分布预测。正态分布认为「-5 倍标准差事件千年一遇」，实际上可能 2-3 年就来一次（2020 年 3 月新冠恐慌中原油跌到负数就是一个例子）。

**应对方案**：
- **压力测试**：不只做历史最大回撤，还要假设「比历史上最差的那天还差 1 倍」时会发生什么。
- **尾部对冲**：用总资金的小部分（如 1%-2%）买入虚值期权作为「黑天鹅保险」。平时这笔钱是净亏损，但极端行情的赔付可以覆盖策略的核心损失。
- **最大杠杆上限**：杠杆是肥尾的放大器。多数灾难来自「高杠杆」遇上「高波动」。建议总杠杆不超过 3 倍。
- **分散到非相关资产**：把部分资金配置到与期货不相关的资产（如债券 ETF）中，降低整体组合的尾部风险。

:::highlight orange
风控终极法则：控制你能控制的（仓位、止损、杠杆、品种分散、策略多样性），接受你不能控制的（市场方向、突发事件、他人的情绪）。任何一个「应该不会发生吧」的风险，如果不设防，总有一天会发生。风控的目的不是让你不亏钱，而是让你在任何极端行情下都能活到明天。
:::

## 完整风控检查清单

每天开机前 / 每周复盘时，逐项检查：

**仓位维度**：
- [ ] 总仓位敞口（所有品种的多头+空头合计）是否在账户的 2-3 倍以内
- [ ] 单品种风险敞口是否 < 总资金的 20%
- [ ] 单板块风险敞口是否 < 总资金的 40%
- [ ] 隔夜持仓是否 < 总资金的 50%（跨夜 = 盲区）

**止损维度**：
- [ ] 服务端硬止损是否已配置且确认生效
- [ ] 每笔交易的最大亏损是否 < 总资金的 2%
- [ ] 单日最大亏损是否有熔断机制（达到 5% 停一天）

**系统维度**：
- [ ] 心跳检测是否正常运行
- [ ] 备用网络 / 备用服务器是否就绪
- [ ] 数据库 / 行情源是否有冗余

**策略维度**：
- [ ] 策略间相关性是否在可接受范围
- [ ] 是否有策略连续亏损超过历史最差水平（退化信号）
- [ ] 交易频率和手续费是否在预期范围内

**心理维度**：
- [ ] 今天是否有重大经济数据发布（非农、CPI、FOMC）
- [ ] 是否有「找个理由干预策略」的冲动
- [ ] 盈亏日志是否已记录
