选对了品种、看准了方向，但如果执行得不好，利润会被滑点和市场冲击吞噬殆尽。执行算法解决的核心问题是：**如何把一个大订单拆成小订单，在给定时间内以最优价格完成交易？**

## 一、为什么要拆分订单？市场冲击成本

### 1.1 一个具体的订单簿穿透案例

假设你想买入 200 手螺纹钢主力合约，当前订单簿如下：

| 档位 | 卖价 | 挂单量(手) | 累计成交量 |
|------|------|-----------|-----------|
| 卖一 | 3550 | 10 | 10 |
| 卖二 | 3551 | 20 | 30 |
| 卖三 | 3552 | 30 | 60 |
| 卖四 | 3553 | 40 | 100 |
| 卖五 | 3554 | 50 | 150 |
| 卖六 | 3555 | 80 | 230 |

如果你直接市价买入全部 200 手：
1. 前 10 手成交在 3550（吃掉卖一）
2. 接下来 20 手成交在 3551（吃掉卖二）
3. 接下来 30 手成交在 3552（吃掉卖三）
4. 接下来 40 手成交在 3553（吃掉卖四）
5. 接下来 50 手成交在 3554（吃掉卖五）
6. 最后 50 手成交在 3555（卖六还剩 30 手）

**成交均价计算**：

$$\text{Avg} = \frac{10×3550 + 20×3551 + 30×3552 + 40×3553 + 50×3554 + 50×3555}{200} = 3553.0$$

**到达价格**（你下单瞬间的 Mid）= 3549.5

**市场冲击** = 3553.0 − 3549.5 = **3.5 元/吨** = 3.5 个 tick

**总冲击成本** = 3.5 元 × 10 吨/手 × 200 手 = **7,000 元**

如果你原本的策略预期每手赚 30 元（共 6,000 元利润），这一次执行就把全部利润吃光还倒亏 1,000 元。这就是为什么执行算法不是「锦上添花」，而是「生死攸关」。

### 1.2 临时性冲击 vs 永久性冲击

学术上把冲击分为两部分：

$$\text{Total Impact} = \underbrace{\text{Temporary Impact}}_{\text{临时性：订单簿回弹}} + \underbrace{\text{Permanent Impact}}_{\text{永久性：信息泄露}}$$

- **临时性冲击**：你的吃单让订单簿暂时空了，但随后做市商会重新挂单，价格会部分回弹。通常在数秒到数分钟内消失。
- **永久性冲击**：市场看到你的大单流向，调整了对「公平价」的判断。这部分冲击不会消失——其他参与者跟随你的方向交易，把价格推到新位置。

**经验比例**：在流动性较好的国内期货主力合约中，临时性冲击约占 **60-70%**，永久性约占 **30-40%**。在流动性差的远月合约，永久性冲击比例会显著上升（因为参与人少，更容易跟风）。

### 1.3 冲击成本模型：线性 vs 平方

**线性模型**（Almgren-Chriss 经典假设）：

$$\text{Impact}(v) = \eta \cdot v$$

其中 $v$ 是订单速率（单位时间成交量），$\eta$ 是冲击系数。该模型假设冲击与速率成正比——简单但低估了大单的真实冲击。

**平方模型**（更接近实际）：

$$\text{Impact}(v) = \eta \cdot v^2$$

平方模型反映了「订单簿深度递减」的事实——越往深吃，每多吃一档冲击越大。学术研究发现，国内商品期货的冲击更接近 $v^{1.5}$ 到 $v^{2}$ 之间。

### 1.4 Almgren-Chriss 核心权衡公式

$$\text{执行成本} = \underbrace{\sigma^2 \cdot T \cdot \text{Var}(\text{剩余订单})}_{\text{波动风险：分太慢会亏}} + \underbrace{\eta \cdot \frac{X^2}{T}}_{\text{市场冲击：分太快会亏}}$$

其中 $X$ 是总订单量，$T$ 是执行时间，$\sigma$ 是价格波动率，$\eta$ 是冲击系数。

> 核心直觉：执行得太快 → 冲击大；执行得太慢 → 价格波动风险大。最优速度是使两者之和最小的点。这是一个典型的「U 形成本曲线」——存在唯一最优执行时间 $T^* \propto \sqrt{\eta / \sigma^2}$。

## 二、TWAP — 时间加权平均价格

TWAP 是最简单的执行算法：在给定时间段内，将总订单均匀拆分执行。这是所有执行算法的「基线」——任何更复杂的算法，首先要能跑赢 TWAP 才有意义。

### 2.1 算法逻辑

1. 确定总数量 N 和执行时间 T（如 200 手螺纹钢，5 分钟）
2. 将 T 等分为 M 个时间片（如 10 片 = 每 30 秒一次）
3. 每个时间片执行 N/M 手
4. 用限价单执行每个时间片，避免市价冲击

### 2.2 数值示例

总单：200 手，5 分钟，10 个 30 秒切片 → 每片 20 手

| 切片 | 时间点 | 执行量 | 累计执行 | 市场均价 |
|------|--------|--------|----------|----------|
| 1 | t=0:30 | 20 | 20 | 3550.2 |
| 2 | t=1:00 | 20 | 40 | 3550.5 |
| 3 | t=1:30 | 20 | 60 | 3550.8 |
| ... | ... | ... | ... | ... |
| 10 | t=5:00 | 20 | 200 | 3552.0 |

**TWAP 成交均价** = 所有切片均价的平均 ≈ **3551.0**

对比一次性市价执行（前面算过 3553.0），**TWAP 节省 2 元/吨** = 4,000 元总成本节约。

### 2.3 TWAP 的优缺点

**优点**：
- 实现极简，几乎不需要历史数据
- 在市场没有明显趋势时（震荡市），执行均价接近时间中点价格
- 适合小单 / 流动性极好的合约

**缺点**：
- **忽视成交量分布**：在成交稀疏时段（如午盘开盘前）也照常下单，冲击放大
- **忽视价格趋势**：若 5 分钟内价格单边上涨，TWAP 会越买越贵
- **可被识别**：均匀的下单节奏很容易被高频做市商识别并前置（front-run）

### 2.4 适用场景

| 场景 | 是否适合 TWAP |
|------|---------------|
| 螺纹钢主力合约，小单（< 50 手） | ✅ 适合 |
| 远月合约，大单 | ❌ 冲击太大，应该选 POV |
| 震荡市，无明显趋势 | ✅ 适合 |
| 单边趋势市 | ❌ 应该选 VWAP 或 IS |
| 套利腿同步执行 | ✅ 适合（同步性强） |

```python
class TWAPExecutor:
    """TWAP 执行器：将订单按时间均匀拆分"""
    def __init__(self, total_qty, duration_min, slices):
        self.total_qty = total_qty
        self.slices = slices
        self.qty_per_slice = total_qty // slices
        self.remaining = total_qty % slices
        self.executed = 0

    def get_next_slice(self):
        if self.executed >= self.total_qty:
            return {"done": True, "quantity": 0}
        qty = self.qty_per_slice
        if self.remaining > 0:
            qty += 1
            self.remaining -= 1
        self.executed += qty
        return {"done": False, "slice_qty": qty, "total_executed": self.executed, "progress": f"{self.executed/self.total_qty*100:.1f}%"}

    def simulate(self, prices):
        avg_price = sum(prices) / len(prices) if prices else 0
        naive_price = prices[0] * 1.01 if prices else 0  # +1%冲击
        savings = (naive_price - avg_price) * self.total_qty
        return {"twap_avg": avg_price, "naive_price": naive_price, "savings": savings}

# 模拟：1万股，5分钟，10个切片
twap = TWAPExecutor(10000, 5, 10)
prices = [20.00, 20.02, 20.01, 20.03, 20.05, 20.04, 20.06, 20.08, 20.07, 20.10]
report = twap.simulate(prices)
print(f"TWAP 均价: {report['twap_avg']:.4f}")
print(f"一次性(模拟+1%冲击): {report['naive_price']:.4f}")
print(f"节约成本: ¥{report['savings']:.2f}")
```

## 三、VWAP — 成交量加权平均价格

TWAP 的改进版：不是按时间均匀拆分，而是按**历史成交量分布**来拆分。以国内期货为例，日盘开盘（9:00 附近）、临近日盘收盘（14:30-15:00）与夜盘开盘（21:00 附近）通常是成交量的三个高峰，午间休市前后和夜盘尾段成交量小——VWAP 会在成交量大的时段下更多的单。

### 3.1 国内期货成交量分布特征

以螺纹钢主力合约为例，一个完整交易日（含夜盘）的成交量分布大致如下：

| 时段 | 成交量占比 | 市场特征 |
|------|-----------|----------|
| 09:00-10:00 | 15-18% | 开盘活跃，资金集中入场 |
| 10:00-11:00 | 12-15% | 盘中相对平稳 |
| 11:00-11:30 | 5-7% | 接近午休，成交量下降 |
| 13:30-14:00 | 8-10% | 午后开盘回升 |
| 14:00-14:30 | 12-15% | 盘中活跃 |
| 14:30-15:00 | 18-22% | 尾盘冲刺，资金博弈加剧 |
| 21:00-22:00 | 12-15% | 夜盘开盘活跃 |
| 22:00-23:00 | 8-12% | 夜盘相对清淡 |

**关键观察**：开盘 30 分钟、午后尾盘 30 分钟、夜盘开盘 30 分钟——这三个时段合计占日成交量 40-50%，是 VWAP 必须重点投放的时段。

### 3.2 VWAP 公式

$$\text{trade\_size}_t = \frac{\text{hist\_volume}_t}{\sum \text{hist\_volume}} \times \text{total\_qty}$$

如果 14:30-15:00 占日成交量 20%，你需要执行的总量是 1000 手，那么这 30 分钟要执行 200 手——而不是按时间均匀分配的 33 手。

### 3.3 VWAP vs TWAP 对比

| 维度 | TWAP | VWAP |
|------|------|------|
| 拆分依据 | 时间均匀 | 历史成交量分布 |
| 数据需求 | 仅需时间 | 需历史成交量曲线 |
| 成交均价 | 时间中点价 | 接近当日真实 VWAP |
| 适用场景 | 震荡市、流动性均匀 | 有明显成交量分布的合约 |
| 抗前置能力 | 弱（节奏可预测） | 中（跟随成交量起伏） |

### 3.4 VWAP 的局限与改进

VWAP 假设「历史成交量分布 = 今日成交量分布」。但当今日有突发消息或资金流异常时，分布会显著偏离历史均值，导致 VWAP 执行效果变差。

**实战改进**：动态 VWAP（Adaptive VWAP）——每 5 分钟用今日实际成交量更新分布预测，动态调整后续切片大小。

## 四、冰山订单 (Iceberg Order)

冰山订单只显示总订单的一小部分（露出水面的「冰山顶端」），其余隐藏。当露出部分被成交后，自动补充新的露出量。

### 4.1 工作机制

假设你想买 1000 手螺纹钢，设置「显示量」为 20 手：
1. 初始挂单：买一价挂 20 手（可见）+ 980 手（隐藏）
2. 20 手被成交后 → 自动补 20 手到买一价
3. 重复 50 次，直到全部 1000 手成交

外部观察者只能看到 20 手的挂单，无法知道你还有 980 手待执行。

### 4.2 用途

- 隐藏大额交易意图，避免引起市场跟风
- 防止其他交易者「抢跑」（front-running）
- 减少大单对市场心理的影响

### 4.3 风险与显示量权衡

**被探测的风险**：高频做市商通过「小额试探单」可以识别冰山订单——他们挂一笔小卖单看你的反应，如果你立即补单，就暴露了。

**显示量的权衡**：
- 显示量太小 → 容易被探测、成交慢
- 显示量太大 → 失去冰山效果，等同于普通限价单

**经验值**：显示量通常设为同时段盘口最优档位挂单量的 **10-20%**。

## 五、实现缺口 (Implementation Shortfall, IS)

### 5.1 定义

$$\text{Implementation Shortfall} = \text{决策价格} - \text{实际执行价格}$$

「决策价格」是你策略给出交易信号时的价格——你「应该」在这个价格成交。「实际执行价格」是真实成交均价。两者之差就是执行过程中的全部隐性成本。

### 5.2 IS 的三段式分解

$$\text{IS} = \underbrace{(P_{arrival} - P_{decision}) \times X}_{\text{延迟成本}} + \underbrace{(P_{exec} - P_{arrival}) \times X}_{\text{交易成本}} + \underbrace{\text{未成交量} \times \text{机会损失}}_{\text{机会成本}}$$

**三个组成部分**：
1. **延迟成本（Delay Cost）**：从决策到首笔成交之间的价格漂移。决策后 100ms 内价格涨了 1 个 tick，这 1 个 tick 就是延迟成本。
2. **交易成本（Trading Cost）**：执行过程中的市场冲击 + 滑点。
3. **机会成本（Opportunity Cost）**：未成交的部分本应赚的钱。如果你打算买 200 手但只成交了 150 手，未成交的 50 手在后续大涨中错过的利润就是机会成本。

### 5.3 数值案例

假设策略在 t=0 给出买入信号，决策价 = 3550，目标 200 手：
- 实际首笔成交在 t=2s，价格 3550.5（延迟成本 = 0.5 × 200 = **100 元**）
- 全部成交均价 3552.0（交易成本 = 1.5 × 200 = **300 元**）
- 其中有 20 手因价格上涨未成交，后续该品种上涨到 3556（机会成本 = 4 × 20 = **80 元**）

**总 IS = 100 + 300 + 80 = 480 元**

这 480 元是从「策略理论利润」到「实盘真实利润」之间的全部损耗。一个优秀的执行算法应该最小化 IS，而非仅最小化其中一项。

### 5.4 IS 的战略意义

IS 是评价执行质量的「金标准」——任何执行算法的优劣最终都用 IS 来衡量。Almgren-Chriss 框架的数学最优解，本质上就是最小化 IS 的方差期望。**实战中建议每日复盘 IS 三段式分解**，找出哪一段成本最高，针对性优化（是延迟太大？冲击太大？还是未成交太多？）。

## 六、POV — 成交量占比算法 (Percent of Volume)

### 6.1 核心思想

不预设执行时间，而是按「市场成交量的一定比例」持续投放订单：

$$\text{trade\_size}_t = \text{POV\_ratio} \times \text{market\_volume}_t$$

例如 POV = 10% 意味着：每当市场成交 100 手，你跟单 10 手。

### 6.2 优缺点

**优点**：
- **自适应市场流动性**：市场活跃时多执行，清淡时少执行——天然匹配成交量分布
- **不需要预测成交量**：相比 VWAP，POV 不依赖历史分布
- **冲击可控**：只要 POV 设得合理（通常 < 20%），单笔对市场冲击恒定

**缺点**：
- **执行时间不确定**：如果市场突然清淡，可能到收盘都没执行完
- **不抗前置**：你的订单节奏与市场节奏高度同步，容易被识别
- **无法应对尾部冲击**：临近收盘未完成的部分被迫市价执行，冲击放大

### 6.3 POV 与 VWAP 的选择

| 场景 | 推荐算法 |
|------|----------|
| 流动性稳定、有历史分布数据 | VWAP |
| 流动性波动大、无法预测 | POV |
| 必须在指定时间完成 | TWAP + 限价 |
| 大单、需要隐藏意图 | 冰山 + POV |

## 七、执行算法选择决策树

```
开始
 ├─ 订单规模 < 最优档位挂单量 10%？
 │   ├─ 是 → 直接限价单（无需算法）
 │   └─ 否 → 进入算法选择
 │
 ├─ 是否有明确时间限制？
 │   ├─ 否 → POV（自适应流动性）
 │   └─ 是 → 进入下一层
 │
 ├─ 订单规模 < 日成交量 5%？
 │   ├─ 是 → TWAP（简单足够）
 │   └─ 否 → 进入下一层
 │
 ├─ 是否有可靠的历史成交量分布？
 │   ├─ 是 → VWAP
 │   └─ 否 → IS（最小化执行缺口）
 │
 └─ 是否需要隐藏意图？
     ├─ 是 → 冰山订单 + 上述算法
     └─ 否 → 直接执行上述算法
```

### 7.1 国内期货实战经验值

| 算法 | 适用订单规模（相对日成交量） | 适合的品种 |
|------|------------------------------|-----------|
| 直接限价 | < 1% | 所有 |
| TWAP | 1-5% | 螺纹钢、铁矿石等主力 |
| VWAP | 3-10% | 所有主力合约 |
| POV | 5-15% | 流动性中等的品种 |
| IS | 10-25% | 大单执行，机构级别 |
| 冰山 + 其他 | 任意 | 需隐藏意图的场景 |

**警告**：当订单规模 > 日成交量 25% 时，**任何算法都无法完全避免冲击**。此时应考虑：拆分到多日执行、切换到流动性更好的合约月份、或与场外大宗交易撮合。

### 7.2 个人量化交易者的实战建议

对于多数个人量化交易者，订单规模通常在日成交量的 0.1-2% 之间，**TWAP + 限价单**已经足够。无需追求复杂的 VWAP/IS 算法——执行算法的复杂度应该匹配你的订单规模。

**关键认知**：执行算法不是越复杂越好。一个简单的 TWAP，如果配合合理的限价单策略和撤单逻辑，往往能跑赢 80% 的「高级」算法。复杂算法的边际收益在小单场景下几乎可以忽略，反而引入了实现风险。

:::highlight orange
执行算法的核心权衡：太快 → 市场冲击大 → 均价变差。太慢 → 时间风险大 → 价格可能朝不利方向大幅波动。没有「最好」的执行算法，只有最适合当前市场流动性和订单规模的执行算法。
:::
