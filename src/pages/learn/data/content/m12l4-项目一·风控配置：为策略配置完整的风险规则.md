## 风控设计原则：从策略到系统的多层防线

在 m7 模块中，你学习了仓位管理和止损的理论。现在你将在系统中为真实策略配置风控规则。

**风控不是策略的「附件」——它是策略的「安全壳」**。好的策略 + 坏的风控 = 归零。差策略 + 好的风控 = 最多亏到止损线。

我们的风控设计遵循「三道防线」原则：

```
第一道：策略自身止损（代码中的 ATR 止损）→ 策略层面
第二道：风控规则（系统自动执行的规则）→ 系统层面  
第三道：紧急熔断（Kill Switch）→ 人工层面
```

> 关联课程：m7l1「仓位管理」→ m7l2「止损」→ m7l4/m7l5「风控陷阱」→ m7l3「凯利公式」

## 步骤 1：查看当前风控状态

打开「风控中心」（/risk），在「概览」Tab 中查看：
- 实时保证金率（当前占用保证金 / 总资金）
- 当日累计盈亏
- 活动委托数
- 当日成交笔数
- 各网关撤单次数

所有这些指标在策略上线前应该是 0 或接近 0——因为还没有开始交易。

## 步骤 2：配置 7 条风控规则

在「规则配置」表中，新增以下规则。假设你的总资金为 **50 万元**，初始只跑螺纹钢一个品种：

**规则 1 —— 单品种持仓上限**
- 作用域：`rb`（策略的品种，按品种级配置避免换月后规则失效）
- 规则类型：`max_position_volume`
- 阈值：**5 手**
- 动作：`reject`（拒绝超限委托）
- 说明：rb 每手保证金约 3500 元，5 手 = 17,500 元保证金，占 50 万资金的 3.5%

**规则 2 —— 单笔最大手数**
- 作用域：全局
- 规则类型：`max_order_volume`
- 阈值：**2 手**
- 动作：`reject`
- 说明：防止误操作一次性下大单（行情软件上不小心多按了一个 0）

**规则 3 —— 活动委托上限**
- 作用域：全局
- 规则类型：`max_active_orders`
- 阈值：**5 笔**
- 动作：`log`（记录日志，不阻止）
- 说明：策略正常运行时不应有超过 5 笔活动委托，超限意味着可能有 bug

**规则 4 —— 日亏损上限**
- 作用域：全局
- 规则类型：`max_daily_loss`
- 阈值：**25,000 元**（总资金的 5%）
- 动作：`halt`（暂停交易）
- 说明：单日亏损超过 5%，立即暂停，人工检查。这是最重要的一道防线

**规则 5 —— 保证金占用率**
- 作用域：全局
- 规则类型：`max_margin_ratio`
- 阈值：**0.30**（30%）
- 动作：`reject`
- 说明：留足 70% 安全垫，应对连续亏损和保证金追加

**规则 6 —— 委托流控**
- 作用域：全局
- 规则类型：`order_flow_limit`
- 阈值：**5 笔/分钟**
- 动作：`reject`
- 说明：防止策略 bug 导致疯狂发单（交易所对撤单频率也有严格限制）

**规则 7 —— 自成交防护**
- 作用域：全局
- 规则类型：`self_trade_prevention`
- 阈值：**启用(1)/禁用(0)**——自成交防护无数值阈值，启用即拦截所有自成交
- 动作：`reject`
- 说明：禁止自己和自己成交（交易所会认定为异常交易行为）

:::highlight blue
**风控规则的数值是怎么算出来的？**

以规则 1（持仓上限 5 手）为例：
- 总资金 50 万，单次最大亏损 2%（即 10,000 元）
- ATR(20) 约 60 元/吨，2 倍 ATR 止损 = 120 元/吨
- 最大可开手数 = 10,000 ÷ (120 × 10) ≈ 8 手
- 保守取 5 手（留安全边际）

规则 4（日亏损上限 25,000 元）的逻辑：
- 5% 的日亏损上限 = 50 万 × 5% = 25,000 元
- 超过了就暂停，不在情绪化中继续交易
- 这个阈值来自 m7l5 的「熔断机制」——一个让你冷静下来的硬止损
:::

```python
# 验证风控规则是否生效：使用事前检查 API
import requests

url = "http://localhost:8000/api/risk/pre-trade/preview"
headers = {
    "Authorization": "Bearer YOUR_TOKEN",
    "Content-Type": "application/json",
}

# 测试 1：发送一个超限的委托（10 手，远超规则 2 的 2 手限制）
payload = {
    "symbol": "rb2610",  # 当前主力合约（示例，实盘按实际主力填写）
    "exchange": "SHFE",
    "direction": "LONG",
    "offset": "OPEN",
    "order_type": "LIMIT",
    "price": 3500,
    "volume": 10,
}
response = requests.post(url, headers=headers, json=payload)
result = response.json()
print(f"允许: {result['data']['allowed']}")
print(f"拒绝原因: {result['data']['rejection_reason']}")
print(f"触发的规则: {result['data']['triggered_rules']}")
# 预期: allowed=False, 原因="超过单笔最大手数限制"

# 测试 2：发送一个正常的委托（1 手，应该通过）
payload["volume"] = 1
response = requests.post(url, headers=headers, json=payload)
result = response.json()
print(f"允许: {result['data']['allowed']}")
# 预期: allowed=True
```

## 步骤 3：理解风控规则层级

风控规则的作用域有三级：

```
策略级（strategy:xxx）> 账户级（account:gateway_name）> 全局（global）
```

如果在策略级和全局级都配置了同一规则类型，**策略级的生效**。这允许你为不同策略设置不同的风控参数。

例如：策略 A 是趋势跟踪（波动大），策略 B 是套利（波动小）。你可以为策略 A 设置更高的持仓上限和日亏损上限，为策略 B 设置更严格的上限。
