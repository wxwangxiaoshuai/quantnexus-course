**交易链路**（程序员视角）：

你（客户端）→ 期货公司（经纪商，即 Broker）→ 交易所撮合引擎 → 清算所

交易所就像一个高并发撮合服务器：买单和卖单进来，按价格优先、时间优先的规则撮合成交。清算所负责确保每笔成交都能结算，是整个系统的信用背书。

**账户资金结构**：

- **总资产** = 可用资金 + 占用保证金 + 持仓浮动盈亏
- **可用资金**：可以用来开新仓的钱
- **占用保证金**：当前持仓锁定的保证金
- **逐日盯市**：每天收盘按**当日结算价**（成交量加权平均价，非最后一笔收盘价）结算一次，盈利划入可用资金，亏损从可用资金扣除。如果可用资金不够补充保证金，会收到**追保通知**（Margin Call）

:::highlight orange
逐日盯市：不像股票那样浮盈浮亏挂着，期货每天都强制结算一次。这意味着你的账户每天都在真实地赚钱或赔钱。
:::

**银期转账：资金如何进出期货账户**：

和股票一样，你不能直接把钱打进期货账户。资金流转路径是：

**银行卡 → 银期转账 → 期货账户**

- 「银」= 银行账户（你在银行的钱）
- 「期」= 期货账户（你在期货公司的交易账户）
- 出入金通过期货公司提供的系统操作（手机 App、交易终端等），到账通常很快（秒到分钟级）
- 出金有时间限制：通常仅限交易日 9:00-15:30 可出金，夜盘时段无法出金

> 量化系统对接时需要注意：如果策略需要根据账户资金动态调整仓位，应以「可用资金」为准，而不是「总资产」——因为占用的保证金不能用来开新仓。

**国内期货交易时段速查**：

各交易所时间略有不同，量化系统的「交易日」概念比日盘 9:00-15:00 更复杂：

| 品种类型 | 日盘 | 夜盘 | 说明 |
|----------|------|------|------|
| 黑色系（螺纹钢/铁矿石） | 9:00-15:00 | 21:00-23:00 | 有夜盘 |
| 有色金属（铜/铝） | 9:00-15:00 | 21:00-次日 1:00 | 夜盘较长，对接 LME |
| 贵金属（黄金/白银） | 9:00-15:00 | 21:00-次日 2:30 | 夜盘最长，对接 COMEX |
| 化工（PTA/甲醇） | 9:00-15:00 | 21:00-23:00 | 有夜盘 |
| 农产品（豆粕/玉米） | 9:00-15:00 | 21:00-23:00 | 有夜盘 |
| 部分农产品（鸡蛋/生猪） | 9:00-15:00 | **无夜盘** | 仅日盘交易 |
| 股指期货 | 9:25-15:00 | **无夜盘** | 与 A 股同步 |

> 量化策略的 K 线合成需要注意：有夜盘的品种，"日线"实际上包含了前一晚夜盘的数据。你在做回测时如果弄错了日线的时间边界，均线计算会歪——这是新手常犯的数据处理错误。

**交易日识别**：对于有夜盘的品种，"一个交易日"通常指「当晚夜盘 + 次日日盘」。比如周一 21:00 开的仓，和周二 9:00-15:00 的交易属于同一个「周一交易日」。
