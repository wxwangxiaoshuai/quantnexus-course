## 三维验证框架：为什么需要三种回测？

在 m6 模块中，你学习了回测的各种陷阱。现在你将在真实系统中同时运行三种验证：

| 验证维度 | 回答的问题 | 系统功能 |
|----------|-----------|----------|
| **标准回测** | 这个策略在历史上表现如何？ | `/backtests` 提交回测任务 |
| **Walk-Forward** | 如果用「当时可用的最优参数」实盘跑，表现如何？ | `/backtests/walk-forward` |
| **敏感性分析** | 滑点/手续费变化时，策略还盈利吗？ | `/backtests/{id}/sensitivity` |

只用标准回测是不够的——因为你在调参时已经「看过」历史数据了。Walk-Forward 模拟了「用过去数据调参、用未来数据验证」的真实场景，是过拟合的最强防线。

> 关联课程：m6l1「回测引擎」→ m6l3「Walk-Forward」→ m6l5「策略稳健性」→ m6l6「Sharpe 显著性」

## 步骤 1：提交标准回测

打开「回测」页面（/backtest），点击「新建回测」，填写以下配置：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 策略模板 | `DonchianBreakoutStrategy` | 上一课创建的模板 |
| 合约 | `rb888.SHFE` | 螺纹钢主力连续合约 |
| K 线周期 | `1d` | 日线 |
| 开始日期 | `2023-01-01` | 样本内起点 |
| 结束日期 | `2024-12-31` | 样本内终点（2 年） |
| 初始资金 | `1,000,000` | 100 万元 |
| 滑点 | `1.0` | 1 个 tick = 1 元/吨 |
| 手续费率 | `0.0001` | 万分之一 |
| 合约乘数 | `10` | 螺纹钢 10 吨/手 |
| 最小变动价位 | `1` | 1 元/吨 |

> **关键参数说明**：
> - **滑点**设为 1.0（而非 0）——每一笔成交比信号价格差 1 个 tick，这是最小滑点
> - **手续费率**万分之一——rb 的手续费约万分之 1，双边收取（开仓+平仓各一次）
> - 这两个值比「默认值」更接近真实交易成本，m6l2 陷阱 4「忽略成本」就是不改这两个参数

点击「提交回测」，等待任务完成。

## 回测结果判断标准（针对趋势跟踪策略）

回测完成后，系统会生成完整的绩效报告。以下是你应该关注的指标和判断标准：

| 指标 | 合格线 | 优秀线 | 我们的目标 |
|------|--------|--------|-----------|
| 年化收益率 | > 10% | > 20% | 15-20% |
| 夏普比率 | > 0.8 | > 1.5 | > 1.0 |
| 最大回撤 | < 30% | < 15% | < 25% |
| 卡玛比率 | > 0.3 | > 1.0 | > 0.5 |
| 胜率 | > 35% | > 45% | 35-40%（趋势策略胜率低是正常的）|
| 盈亏比 | > 1.5 | > 2.5 | > 2.0 |

> **注意**：趋势跟踪策略天然胜率低（35-40%）、盈亏比高（> 2.0）。如果胜率 > 50%，反而要警惕——可能是过拟合或使用了未来数据。

分析回测报告时，除了看指标，还要看：
- **资金曲线形态**：是否稳步上升？有没有长期横盘回撤？
- **月度收益分布**：12 个月中几个月盈利？是否有连续亏损月？
- **交易记录**：最大单笔盈利和亏损是多少？有没有异常交易？
:::

## 步骤 2：Walk-Forward 滚动验证

在回测页点击「Walk-Forward」Tab：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 训练窗口 | `252` 天 | 用 1 年数据调参 |
| 验证窗口 | `63` 天 | 用 1 季度数据选最优参数 |
| 测试窗口 | `63` 天 | 真正的样本外测试 |
| 步长 | `63` 天 | 每次向前滚动 1 季度 |

Walk-Forward 的输出：
- 每个窗口的样本外资金曲线
- 整体样本外 Sharpe
- 对照：样本内 Sharpe vs 样本外 Sharpe

**判断标准**：
- ✅ 样本外 Sharpe > 0.5 → 策略在未见过的数据上仍然有优势
- ✅ 样本外 / 样本内 Sharpe > 0.5 → 过拟合程度可控
- ❌ 样本外 Sharpe < 0 → 策略完全失效，需要重新设计

## 步骤 3：参数敏感性分析

在回测结果页点击「敏感性分析」：

| 参数 | 测试范围 |
|------|----------|
| 滑点 | 0.5, 1.0, 1.5, 2.0, 3.0 |
| 手续费 | 0.00005, 0.0001, 0.0002, 0.0005 |

观察：当滑点从 1.0 变到 3.0 时，Sharpe 变化多少？手续费翻倍时策略是否仍然盈利？

**好策略的标准**：参数在合理范围内变化时，策略绩效不应剧烈波动。如果滑点从 1.0 变到 1.5 就让策略从大赚变亏损，说明策略对交易成本极度敏感——实盘几乎不可能盈利。

```python
# 参数优化配置（在回测页的「参数优化」Tab 中使用）
# 不要直接在回测前做参数优化——先做标准回测确认策略逻辑正确

# 网格搜索参数范围
optimize_config = {
    "objective": "sharpe_ratio",       # 优化目标
    "algorithm": "bf",                 # 暴力搜索（网格搜索）
    "param_ranges": {
        "breakout_period": [20, 30, 40, 50, 60],
        "adx_threshold": [20, 25, 30],
        "atr_stop_mult": [1.5, 2.0, 2.5],
    }
}
# 共 5 × 3 × 3 = 45 组参数组合
# 注意：优化结果必须在 Walk-Forward 中验证，否则只是过拟合
# 最优参数只在「训练段」上找，然后用「测试段」验证——这才是可信的
```
