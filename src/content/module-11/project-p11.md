# ML 陷阱复盘

> 预计 **45 分钟** · 难度 **专家** · 核心互动：前视偏差挑战

## 背景

ML 策略最容易死在 **特征泄露** 和 **前视偏差**——模型在回测里「偷看未来」，实盘立刻失效。本项目的核心是建立 **上线前检查习惯**。

## 实验步骤

### 步骤 1：完成前视偏差挑战

1. 阅读 [L11-04 ML 策略的实战陷阱](/curriculum/11/L11-04)
2. 完成该课中的 **前视偏差挑战** 互动（`lookAhead`；[L06-02](/curriculum/6/L06-02) 亦有同类练习可对照）
3. 记录错题与正确思路

### 步骤 2：防泄露检查表

编写 **≥8 项** 检查表，至少包含：

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 1 | 标签是否使用未来信息 | |
| 2 | 特征计算窗口是否严格 ≤ 决策时刻 | |
| 3 | 训练/测试是否时间顺序切分 | |
| 4 | 是否使用 Purged/Embargo CV | |
| 5 | 全局归一化是否只在训练集 fit | |
| 6 | 是否排查「未来函数」API | |
| 7 | 高 in-sample R² 是否触发人工复核 | |
| 8 | 特征是否经济可解释 | |

### 步骤 3：举例

写 **1 个**「看起来合理但泄露」的特征例子（如用「明日涨跌幅」构造的伪特征）。
