## ML 策略的失败率远超传统策略——为什么？

在量化交易中，ML 策略的样本外衰减速度比传统策略快 3-5 倍。这不是因为 ML 不好——是因为 ML 太容易被**误用**。

> 关联课程：m11l1「线性模型」→ m11l2「特征工程」→ m11l3「树模型」→ m6l6「Sharpe 显著性」

## 陷阱 1：标签泄漏——ML 在金融中的头号 Bug

**标签泄漏** = 你的特征 X 里不小心包含了未来信息（标签 Y 的衍生信息），模型在训练时「看到了不该看的」。

**最常见的标签泄漏场景**：

| 场景 | 错误做法 | 正确做法 |
|------|----------|----------|
| 用 t 日的收盘价做特征 | `feature = close[t]` — 没问题 | ✅ |
| 用 t 日的收益率做特征 | `feature = ret[t]` — 没问题 | ✅ |
| 用 t+1 日的信息做特征 | `feature = ret[t+1]` — **泄漏！** | ❌ |
| 全量数据标准化 | 用全部数据算 mean/std → 包含未来信息 | 只用训练集算 mean/std |
| PCA 降维 | 对全部数据做 PCA → 主成分包含了未来结构 | 只用训练集拟合 PCA |
| 缺失值填充 | 用全部数据的均值填充 → 未来数据参与计算 | 只用训练集的均值填充 |

**标签泄漏的检测方法**：

1. 如果模型在训练集上 R² > 0.3，但你在金融数据上几乎不可能达到这个水平 → 99% 有标签泄漏
2. 随机打乱标签 Y（破坏任何真实的 X→Y 关系），重新训练。如果 R² 仍然很高 → 100% 有标签泄漏
3. 逐特征检查：删除某个特征后 R² 暴跌 → 可能是这个特征有泄漏（也可能是它真的有效，需要人工判断）

:::highlight orange
## 陷阱 2：Purged K-Fold——金融 ML 的必修课

m11l2 讲了时序交叉验证（TimeSeriesSplit），但金融 ML 还有一个额外的要求：**Purge**。

**问题**：时序交叉验证虽然保证了「训练在验证之前」，但训练集的最后几天和验证集的最初几天在时间上相邻——它们的收益率可能高度相关（波动率聚集）。这意味着模型在验证集上的表现被高估了。

**Purged K-Fold**：在训练集和验证集之间插入一个「隔离期」（Purge Gap），丢弃这几天数据。

```
标准 TimeSeriesSplit：
Fold 1: [train: Jan-Mar] → [test: Apr]

Purged TimeSeriesSplit：
Fold 1: [train: Jan-Mar] → [Purge: Apr 1-5] → [test: Apr 6-30]
                                        ↑
                              丢弃 5 天隔离期，防止信息泄漏
```

**隔离期长度**：通常设为 5-10 个交易日，取决于你的持仓周期。

> 如果你不做 Purge，你的验证集 R² 可能被高估 20-50%。这是回测 ML 策略时最隐蔽的陷阱之一。
:::

## 陷阱 3：ML 策略的 Alpha 衰减为什么更快？

传统策略（双均线、唐奇安通道）基于经济逻辑（趋势惯性、均值回归），即使很多人用也不会完全失效——因为趋势惯性是市场的基本属性。

ML 策略基于**统计规律**（在历史数据上学到的模式），当市场结构变化时，统计规律比经济逻辑更容易失效。

**ML 策略的衰减信号**：
- 模型预测的置信度分布越来越集中（所有预测都在 ±0.5% 之间）→ 模型找不到信号了
- 特征重要性排名剧烈变化（每个月 Top 5 特征完全不同）→ 模型不稳定
- 样本外 R² 在 3 个月内从 0.03 降到 0.005 → Alpha 衰减

## 陷阱 4：ML 预测概率 ≠ 仓位大小

ML 模型输出一个预测值（如「下期收益率 = +0.02」），很多新手直接把它映射为仓位大小：

```python
# 错误做法
position = prediction * leverage  # prediction 越大，仓位越大
```

**问题**：ML 模型的预测值在样本外的分布和样本内可能完全不同。样本内 prediction 范围是 [-0.03, +0.03]，样本外可能是 [-0.01, +0.01]——你的仓位会大幅缩水。

**正确做法**：

1. 用预测值的**排名**而非绝对值（做多 Top 20%，做空 Bottom 20%）
2. 或者用预测值的**分位数**映射仓位（预测值处于历史 90% 分位 = 满仓做多）
3. 仓位大小由风控规则决定（ATR 仓位法），不由 ML 预测值决定

## ML 策略的正确使用姿势

1. **用 ML 做因子，不要用 ML 做策略**。ML 的输出是一个信号（因子值），策略的仓位管理、止损、风控仍然由传统规则决定
2. **永远用排名，不要用绝对值**。ML 擅长排序（哪个品种更好），不擅长预测精确的收益率
3. **保持简单**。线性模型 + Ridge 正则化 > 复杂神经网络。金融数据信噪比极低，简单模型反而更稳健
4. **时序交叉验证 + Purge 隔离期**。这是 ML 策略回测的最低标准，不做 Purge 的结果不可信
5. **样本外锁定**。ML 模型训练完成后，在样本外数据上永远不要再修改任何参数。如果样本外表现不好，回到样本内重新训练——但这是一次「新的实验」，不是「修改原来的实验」
6. **ML 策略的仓位上限 = 传统策略的 50%**。ML 策略的衰减速度更快，在你不确定它是否还有效之前，用更小的仓位跑

> ML 在量化交易中不是「银弹」——它是一把锋利的刀，用好了可以切出传统方法看不到的 Alpha，用不好会切到自己。关键不是模型有多复杂，而是你对金融数据的理解有多深。
:::
