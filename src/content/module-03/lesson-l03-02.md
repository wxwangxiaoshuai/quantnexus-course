**K 线 = 时间窗口聚合**：把一段时间内（1 分钟、5 分钟、日线……）的所有 tick 聚合成 4 个价格点：

- **O（Open）**：该时间段第一笔成交价
- **H（High）**：该时间段最高成交价
- **L（Low）**：该时间段最低成交价
- **C（Close）**：该时间段最后一笔成交价
- **V（Volume）**：该时间段总成交量

:::highlight orange
主力合约换月陷阱：每隔几个月，流动性会从旧合约（如 rb2501）切换到新合约（rb2505）。如果你的数据直接拼接两个合约的价格，会出现巨大跳空，导致均线、指标全部失真。正确做法是使用「复权」后的连续合约数据。
:::

**K 线周期的选择策略**：

你用多长时间的 K 线作为信号输入，决定了策略的持仓周期和交易频率。没有「最好」的周期，只有「适合你策略」的周期：

| 周期 | 每根 K 线时间 | 适用策略 | 典型缺点 |
|------|-------------|----------|----------|
| 1 分钟 | 60 秒 | 高频/剥头皮 | 噪音极大，手续费高，假信号多 |
| 5 分钟 | 5 分钟 | 日内短线 | 对网络延迟敏感 |
| 15 分钟 | 15 分钟 | 日内波段 | 信号次数减少 |
| 30 分钟 | 30 分钟 | 日内+隔夜 | 日内的止损空间较大 |
| 1 小时 | 60 分钟 | 短期趋势跟踪 | 每天只有 3-4 根（日盘 4 小时） |
| 日线 | 1 个交易日 | 中长期趋势跟踪 | 反应缓慢，利润回吐大 |
| 周线 | 1 周 | 宏观配置 | 单品种一年只有 52 根 K 线，信号太少 |

> 量化策略中常见的组合：用日线确定趋势方向（大周期定调），用 15 分钟或 30 分钟 K 线寻找具体入场点（小周期执行）。这叫「多周期确认」——大周期的趋势 + 小周期的信号共振，可以减少假信号。

**用代码表示多周期过滤**：

```python
def multi_tf_signal(daily_bars, hourly_bars):
    """日线定方向，小时线找入场"""
    daily_ma20 = sma([b.close for b in daily_bars], 20)[-1]
    hourly_ma5 = sma([b.close for b in hourly_bars], 5)[-1]
    price = hourly_bars[-1].close

    # 日线在 MA20 之上（大趋势向上）+ 小时线价格突破 MA5（小时级别确认）
    if price > daily_ma20 and price > hourly_ma5:
        return 'BUY'
    return 'HOLD'
```

:::highlight blue
换月的正确姿势——复权（Adjustment）：

复权就是「把不同合约的价格修正为可连续使用的序列」。这里沿用股票的「前复权/后复权」术语，方向一定要记准（很多人记反，拿错方向的数据去 vn.py / 数据商接口会直接踩坑）：

**前复权（Forward Adjustment）**：以**当前（最新）合约价格为基准，当前价保持不变，反向修正历史价格**。假设 rb2501 最后交易日收盘 3400，切换到 rb2505 当日收盘 3450，价差 = 3450 - 3400 = +50。则把 rb2501 及更早的所有历史价格统一 +50，让它们与新合约衔接。

优点：当前价格就是真实市价，最贴近实盘，适合实时策略与看图。缺点：越早的历史价格越偏离当年的真实成交价，甚至可能出现负价。

**后复权（Backward Adjustment）**：以**最早（最旧）合约价格为基准，最早价保持不变，向后修正后续所有价格**。历史起点是真实成交价，越新的价格被调整得越多。

优点：历史起点真实，适合计算长期累计收益率。缺点：当前价格不等于真实市价，不能直接拿去实盘下单。

> 记忆口诀：**「前」复权钉住当前价改历史，「后」复权钉住最早价改后面**。CTA 连续合约回测最常用前复权（保证近端价格真实）。

> 在 QuantNexus 或 vn.py 中，换月操作通常在数据维护脚本中完成。回测引擎读取的是已复权的连续合约数据。你的策略代码不需要处理换月逻辑——这是数据层的责任。
:::

**数据清洗：量化系统的第一道防线**：

实盘行情数据不会像教材中的干净。你需要了解以下常见脏数据类型及其处理方式：

1. **缺失数据**：某分钟没有产生 tick → K 线缺失一根。处理方法：用前一根 K 线的收盘价填充（forward fill），或标记为 NaN 让策略自行跳过。

2. **价格异常**：某 tick 的成交价突然是正常价格的 10 倍（撮合引擎 bug 或数据录入错误）。处理方法：设置价格波动阈值（如当日涨跌停板的 120%），超出的 tick 直接丢弃。

3. **时间戳乱序**：tick A 的时间戳晚于 tick B，但 A 先到达。处理方法：在 tick 进入策略前做排序或使用事件时间而非到达时间。

4. **成交量为 0 的价格**：集合竞价产生的开盘价 tick 有时成交量 = 0，K 线聚合时应排除。

5. **重复 tick**：网关重连后重发的 tick。处理方法：用 (symbol, datetime, lastPrice, volume) 四元组去重。

> 一个实用的数据质量检查脚本（在回测开始前运行）：

```python
def check_data_quality(bars: list[Bar]) -> dict:
    issues = {
        'null_count': sum(1 for b in bars if b.close is None),
        'zero_volume': sum(1 for b in bars if b.volume == 0),
        'price_jumps': 0,
        'out_of_order': sum(1 for i in range(1, len(bars))
                           if bars[i].datetime <= bars[i-1].datetime),
    }
    return issues  # 回测前检查：任何一项 > 0 都应该人工检查
```

**成交量与价格的关系：量价分析入门**：

成交量和价格的关系就像汽车的「油门」和「速度」——光看速度不知道动能从哪来，光看油门不知道车跑得多快。两者结合才是完整的。

**量价四象限**：

| 价格 | 成交量 | 含义 |
|------|--------|------|
| 涨 | 放量 | 多头强势，有持续动力（量价配合） |
| 涨 | 缩量 | 上涨动力减弱，可能是尾声（量价背离） |
| 跌 | 放量 | 空头强势，恐慌性抛售 |
| 跌 | 缩量 | 下跌动能减弱，卖盘枯竭 |

> 量价背离是一个经典的危险信号：价格创新高但成交量却缩小——说明推动价格上涨的买盘在减少，顶部的概率增加。在量化策略中，可以把成交量确认作为开仓信号的附加条件。
