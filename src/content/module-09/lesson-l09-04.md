## 课程讲了如何研发策略，但没讲什么时候该停掉它

大部分量化教程教你「怎么做出好策略」，但几乎没人教你「怎么判断一个策略该退役了」。而在真实交易中，**策略下架决策和策略研发同等重要**——你不可能永远跑同一个策略。

**策略和人一样，有生命周期**：出生（研发）→ 成长（优化）→ 成熟（稳定盈利）→ 衰退（Alpha 衰减）→ 退役（下架）。每个策略最终都会走到退役这一步，你只是选择在它还能为你赚钱的时候体面地退役，而不是亏损到不得不退。

> 关联课程：m8l3「策略退化与生命周期」

## 策略下架决策框架：红黄绿三档

**🔴 强制下架（立即停止，无需讨论）**：

- 单日亏损超过 5% → 可能策略代码有 bug 或市场发生极端事件
- 策略代码出现严重错误（除零、数组越界、信号逻辑错误）
- 策略品种退市 / 流动性枯竭（如 2021 年动力煤被暂停交易）
- 交易所规则变更导致策略逻辑不再有效（如手续费调整、保证金比例变更）

**🟡 下架评估（满足 2 项以上，进入 1 个月评估期）**：

- 滚动 30 日 Sharpe < 0.3，持续 3 个月
- 回撤超过历史最大回撤的 1.5 倍
- 实盘收益持续低于回测预期的 50%，超过 3 个月
- 策略间相关性从 0.3 上升到 0.7（策略失去了分散化价值）
- 有一个更好的替代策略，旧策略的边际贡献为负

**🟢 正常观察（继续运行，正常监控）**：

- 滚动 Sharpe 在 0.3-0.8 之间 → 继续观察
- 滚动 Sharpe > 0.8 → 正常

:::highlight blue
**策略下架 ≠ 策略失败**。这是新手最容易犯的认知错误。

Alpha 衰减是所有量化策略的宿命——你的策略不是在「失败」，而是在经历正常的生命周期。就像软件产品会下线、旧版本会被替代一样，策略下架是正常的资产管理行为。

优秀的策略管理者不是「从不失败」的人，而是「能在策略还赚钱时预判到它即将失效，并提前准备好替代方案」的人。
:::

## 策略下架的实际操作流程

当你决定下架一个策略时，按以下步骤操作：

**步骤 1：停止策略**
- 在策略工作台（/strategy）点击「停止（Stop）」
- 策略不再生成新信号

**步骤 2：处理现有持仓**
- 评估现有持仓的风险——如果风险可控，不急于平仓（避免恐慌性滑点）
- 如果风险不可控（如策略逻辑已失效），以市价平掉所有持仓

**步骤 3：撤销挂单**
- 在交易页（/trading）撤销该策略的所有未成交委托

**步骤 4：记录下架原因**
- 写一份简短的下架报告（模板见下）

**步骤 5：启动替代策略**
- 从你的「策略管道」中选择一个已验证的候选策略
- 先小资金运行 1 周，确认无异常后逐步加仓

## 策略下架报告模板

```
策略名称：DonchianBreakout_rb
下架日期：2025-XX-XX
运行时长：X 个月
累计收益：+X%
下架原因（选择一项）：
  □ 强制下架：_________________
  □ 正常退役：_________________
替代策略：_________________
经验教训：_________________
```

:::highlight orange
**区分「正常波动」和「策略永久失效」是专家和新手的核心区别**。

| 现象 | 正常波动 | 永久失效 |
|------|----------|----------|
| Sharpe 从 1.2 降到 0.8 | ✅ 正常 | ❌ 不是失效 |
| Sharpe 从 1.2 降到 0.1（持续 3 个月）| ❌ 不是正常 | ✅ 可能失效 |
| 单月亏损 8%（历史最大回撤 15%）| ✅ 正常 | ❌ 不是失效 |
| 回撤突破历史最大回撤的 1.5 倍 | ❌ 不是正常 | ✅ 可能失效 |
| 连续 5 天没有信号 | ✅ 可能是震荡市 | ❌ 不是失效 |
| 连续 3 个月没有信号 | ❌ 不是正常 | ✅ 可能品种已退市 |

**核心原则**：不要因为短期的正常波动就下架策略（那是情绪化交易），也不要因为舍不得而留着一个已经永久失效的策略（那是沉没成本谬误）。
:::
