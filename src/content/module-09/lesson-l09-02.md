实盘不是「部署后就不用管了」。实盘是持续运营——每天检查、每周复盘、每月评估。本节给你一份可以直接拿来用的实盘运维清单。

## 每日检查清单（开盘前 30 分钟）

**系统状态**：
- [ ] 行情源是否正常连接（检查是否有断线/延迟）
- [ ] 交易网关是否正常登录
- [ ] 所有策略进程是否正常运行
- [ ] 数据库连接是否正常
- [ ] 风控进程/守护进程是否正常

**盘前数据**：
- [ ] 昨日结算价是否已更新
- [ ] 今日是否有合约到期/交割
- [ ] 是否有品种调整保证金/手续费（交易所公告）
- [ ] 是否有重大经济数据今日发布（非农、CPI、FOMC、EIA 等）

**策略状态**：
- [ ] 每个策略的当前持仓是否与预期一致
- [ ] 是否有未撤销的失效委托
- [ ] 账户总风险敞口是否在限额以内

## 盘中实时监控（持续）

**关键仪表盘指标**：
- 账户总权益与可用资金
- 各品种实时浮盈浮亏
- 各策略信号生成频率（是否突然增多/减少）
- 委托成交率（是否有异常的大量拒单/部分成交）
- CPU / 内存 / 网络延迟

**告警阈值**（设置自动通知）：
| 指标 | 黄灯阈值 | 红灯阈值 |
|------|----------|----------|
| 单日浮亏 | > 2% | > 5% |
| 盘中回撤 | > 历史MDD的70% | > 历史MDD |
| 行情延迟 | > 500ms | > 2s |
| 连续拒单 | > 3次 | > 5次 |
| CPU占用 | > 80% | > 95% |

## 每周复盘（周末 1-2 小时）

- [ ] 本周收益 vs 预期收益对比
- [ ] 本周胜率 vs 历史胜率对比
- [ ] 是否有策略触发退化信号
- [ ] 是否有异常大额亏损的单笔交易
- [ ] 手续费是否在预算范围内
- [ ] 策略运行日志是否无异常错误
- [ ] 更新策略的滚动 Sharpe / Sortino / MDD

## 每月评估（月末）

- [ ] 各策略月度绩效表（收益、Sharpe、MDD、胜率）
- [ ] 策略间相关性矩阵是否有显著变化
- [ ] 是否需要调整策略间的资金分配（再平衡）
- [ ] 策略管道（Pipeline）中的候选策略进度更新
- [ ] 是否需要归档已经退化的策略

## 资金管理与再平衡

实盘中，不同策略的收益率不同，自然会导致资金分配偏离最初的设定。**再平衡**就是定期重新调整各策略的资金分配，使权重恢复到目标比例。

**再平衡频率**：
- 每天再平衡：交易成本高，不推荐
- 每周再平衡：适合高频策略
- 每月再平衡：最适合大多数中低频策略
- 阈值再平衡：当任一策略权重偏离目标 5%+ 时才调整

**再平衡计算公式**：
$$
\text{Adjustment}_i = \text{TargetWeight}_i \times \text{TotalEquity} - \text{CurrentValue}_i
$$

正的 Adjustment 需要追加资金，负的需要撤回。注意：再平衡本身会产生交易成本，需在计算中考虑。

## 实盘日志规范

量化交易者需要养成记录日志的习惯。推荐格式：

```
[2024-01-15] 实盘日报
- 账户权益：¥523,000（+0.8%）
- 最大浮亏：-1.2%（盘中）
- 活跃策略：3 / 3 正常运行
- 异常事件：09:31 行情延迟 800ms，已自动切换备用线路
- 今日操作：无人工干预
- 备注：明天 20:30 美国 CPI 发布，已降低夜盘仓位至 30%
```
