# 实盘上线 SOP

> 预计 **60 分钟** · 难度 **高级** · 文档型项目（无专用互动）

## 背景

回测赚钱 ≠ 实盘赚钱。职业团队用 **SOP（标准作业程序）** 把「可以上线」变成可检查、可审计的清单。本模块课节已提供素材，你的任务是把它们 **整合成一份自己的 SOP 草稿**。

## 交付物 1：运维检查清单

复制以下模板并 **逐项填写**（可删改，但需保留核心类别）：

```markdown
## 上线前检查清单

### 数据
- [ ] 数据源与回测一致
- [ ] 主力合约/换月规则已确认
- [ ] ...

### 策略
- [ ] 样本外 / Walk-Forward 通过
- [ ] 无未来函数
- [ ] ...

### 风控
- [ ] 单笔/日亏损上限
- [ ] 最大持仓/保证金占用
- [ ] ...

### 监控与告警
- [ ] 心跳 / 断线告警
- [ ] PnL 异常告警
- [ ] ...

### 人工复核
- [ ] 模拟盘运行 ≥ N 天
- [ ] 负责人签字：________
```

## 交付物 2：下架决策标准

至少定义 **2 个可量化** 下架条件，例如：

- 滚动 30 日 Sharpe < ___
- 最大回撤超过回测期的 ___ 倍
- 连续 ___ 日未产生预期信号
