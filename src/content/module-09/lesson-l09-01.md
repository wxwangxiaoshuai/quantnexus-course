**阶段一：策略研发**
在历史数据（样本内）上开发策略，调参，建立初步认知。警惕过拟合。

**阶段二：样本外测试**
用策略「没见过的」数据测试（时间切分：2020 之前开发，2021-2023 验证）。样本外表现比样本内差是正常的，差很多则可能过拟合。

**阶段三：模拟盘**
用真实行情跑策略，但不下真实资金。观察实际成交、滑点、系统延迟。

**阶段四：小资金实盘**
用极小仓位跑真实账户。目的不是赚钱，而是验证整个系统没有 bug。

**阶段五：正式实盘**
逐步加大资金，持续监控策略表现是否与预期一致。

:::highlight orange
实盘 vs 回测的最大差异：滑点比预期大、偶尔部分成交、网络偶发断线、夜盘数据异常……这些都是回测无法复现的真实世界复杂性。
:::

## 最小可行实盘检查清单

在投入真实资金之前，按此清单逐项确认。每一项都是「上线」的必要条件——如果你在某个项目上犹豫了，就暂停上线。

**策略层面（反违规 + 无重大退化）**：
- [ ] 策略在至少 2 年的样本外数据上跑出了正向收益（不要求 Sharpe > 2，要求 > 0）
- [ ] 参数敏感性分析中，核心参数变化 ±20% 的结果不亏损
- [ ] 蒙特卡洛模拟 1000 次，盈利概率 > 70%
- [ ] 在 2+ 个品种上（即使主要交易品种只有 1 个）样本外测试结果为正
- [ ] 策略没有使用任何未来函数（前视偏差检查完成）

**风控层面（可以停止亏损）**：
- [ ] 单笔交易最大亏损 ≤ 总资金的 2%（无论策略逻辑多强，单笔止损必须硬编码）
- [ ] 单日亏损熔断线 = 总资金的 5%
- [ ] 服务端硬止损已配置且手动测试通过（在模拟盘中触发一次看是否立刻平仓）
- [ ] 最差情况下的连续亏损序列（历史最大连亏次数 × 1.5）下，账户剩余资金 > 50%

**系统层面（可以持续运行）**：
- [ ] 在模拟盘上连续运行 1 周以上无崩溃、无内存泄漏
- [ ] 网络中断恢复后策略能自动恢复正常运行
- [ ] 每天开盘和收盘时各检查一次「系统是否健康」（心跳机制）
- [ ] 日志完整记录了每次委托的状态变更（已提交 → 部分成交 → 全部成交 / 撤销 / 拒单）

**数据层面（策略进的是什么数据）**：
- [ ] 行情源的数据完整性验证通过（至少 3 天对比：数据源的 OHLC vs 交易所官网）
- [ ] 换月复权逻辑经手动验算正确
- [ ] 持仓成本和手续费参数与经纪商合同一致

**运维层面（出问题知道怎么办）**：
- [ ] 紧急停机流程 SOP 已打印/保存并可离线访问
- [ ] 经纪商客服电话已保存在手机中（不是只有邮箱——紧急时需要电话）
- [ ] 备用网络（手机热点）已测试可用

> 如果这份清单看起来很长——它是的。实盘不是回测，真正的钱在市场上是有对手的。这份清单的每一项都是从真实爆仓故事中总结出来的。
